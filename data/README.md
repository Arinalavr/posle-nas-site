# data/

Источник правды для контента сайта. Все статичные данные — фото, истории, цитаты — лежат здесь и подгружаются в компоненты.

## Файлы

### `items.json`
Массив вещей. Одна вещь = одна запись. Несколько вещей у одного информанта = несколько записей с одинаковым `informant_id`.

```json
{
  "id": "item_R02_gantel",
  "informant_id": "R02",
  "photo": "/photos/R02/gantel.jpg",
  "title": "Гантели",
  "story": "Куплены на барахолке в Будве у уехавшей семьи...",
}
```

Поля:
- `id` — уникальный, формат `item_<informant_id>_<short_name>`
- `informant_id` — R01–R17
- `photo` — путь от `public/`, начинается с `/photos/`
- `title` — короткое название для карточки (1–3 слова)
- `story` — 3–5 предложений (из `документа историй по вещам/`)

### `quotes.json`
Массив цитат. Не привязаны к конкретной вещи — распределены по эмоциональной арке.

```json
{
  "id": "q001",
  "informant_id": "R01",
  "text": "Когда я приехала, я не понимала где я нахожусь...",
  "arc": "early_emigration"
}
```

Поля:
- `id` — уникальный
- `informant_id` — R01–R17
- `text` — цитата дословно
- `arc` — `early_emigration` | `things` | `feelings` | `loss_of_friends`
- `order` — предложенный порядок в основной выставке

### `exhibition.json`
Монтажная последовательность основной выставки. Сейчас там только цитаты; позже между ними можно вставить вещи.

```json
[
  { "type": "quote", "id": "q001" },
  { "type": "item", "id": "item_R02_ganteli" }
]
```

## Откуда что приходит

- **`items.json`:** фото из `public/photos/R##/` + истории вещей, которые нужно дописать в поле `story`
- **`quotes.json`:** отобранные цитаты из `../Цитаты отобранные.md`, с тегом `arc` и порядком `order`
- **`exhibition.json`:** ручной порядок элементов на главной странице
