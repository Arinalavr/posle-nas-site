# data/

Источник правды для контента сайта. Здесь лежат статичные данные выставки: вещи, цитаты и порядок элементов на главной странице.

Фотографии хранятся отдельно в `public/photos/`, логотип — в `public/logo/`.

## Файлы

### `items.json`

Массив вещей для карточек и фотографий на выставке.

Сейчас в файле 19 записей. Одна вещь = одна запись. Если у одного информанта несколько вещей, они идут отдельными записями с одинаковым `informant_id`.

Пример:

```json
{
  "id": "item_R14_kamni",
  "informant_id": "R14",
  "slug": "kamni",
  "title": "Камушки с пляжа",
  "photo": "/photos/R14/kamni.jpg",
  "photo_alt": "Камушки с пляжа",
  "story": "В новой квартире всё было слишком стерильно...",
  "order": 1,
  "featured": true
}
```

Поля:

- `id` — уникальный идентификатор вещи, формат `item_R##_short_name`.
- `informant_id` — номер информанта.
- `slug` — короткое латинское имя вещи.
- `title` — название вещи на сайте.
- `photo` — путь к фотографии от папки `public/`, всегда начинается с `/photos/`.
- `photo_alt` — короткое описание фотографии для доступности.
- `story` — самостоятельная история вещи для карточки.
- `order` — порядок на странице вещей.
- `featured` — служебный флаг для вещей, выбранных в текущую версию выставки.

### `quotes.json`

Массив цитат для основной выставки.

Сейчас в файле 30 цитат. Цитаты не привязаны к конкретным вещам и распределены по эмоциональной арке.

Пример:

```json
{
  "id": "q001",
  "informant_id": "R04",
  "text": "Пока сидим в Черногории, пока не выгонят",
  "arc": "early_emigration",
  "order": 1
}
```

Поля:

- `id` — уникальный идентификатор цитаты.
- `informant_id` — номер информанта.
- `text` — текст цитаты без кавычек.
- `arc` — смысловой тег цитаты.
- `order` — порядок цитаты внутри базы.

Возможные значения `arc`:

- `early_emigration` — временность, ранняя эмиграция, неустойчивость.
- `things` — вещи, багаж, предметы, обживание.
- `feelings` — ощущения от эмиграции и новой жизни.
- `loss_of_friends` — расставания, дистанция, потеря связей.

На сайте эти теги показываются более мягкими русскими подписями:

- `early_emigration` → `временность`
- `things` → `вещи`
- `feelings` → `ощущения`
- `loss_of_friends` → `расставания`

### `exhibition.json`

Монтажная последовательность главной страницы.

Сейчас в файле 49 элементов: цитаты и вещи в том порядке, в котором они появляются в маршруте выставки.

Пример:

```json
[
  { "type": "quote", "id": "q001" },
  { "type": "item", "id": "item_R14_kamni" }
]
```

Поля:

- `type` — тип элемента: `quote` или `item`.
- `id` — идентификатор записи из `quotes.json` или `items.json`.

## Как это используется на сайте

- Главная страница читает `exhibition.json` и по `id` подтягивает нужные цитаты и вещи.
- Страница вещей читает `items.json` и показывает все карточки по порядку.
- Подписи смысловых тегов цитат задаются в `src/lib/content.ts`.
- Размеры фотографий для раскладки также задаются в `src/lib/content.ts`, чтобы сохранять исходные пропорции снимков.
